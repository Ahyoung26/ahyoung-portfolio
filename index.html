🧭 What Kind of Planner

문제를 기능이 아닌
'사용 흐름과 구조'로 해결하는 기획자

데이터 기반으로 사용자 행동과 이탈 지점을 분석하고,
UX 흐름, 정책, 데이터 구조를 통합적으로 설계해왔습니다.

특히 AI 서비스 영역에서는 가상인간과 LLM 챗봇 프로젝트를 통해

사용자 입력 → 판단 → 응답으로 이어지는
대화형 구조를 설계해왔으며,

이 경험을 바탕으로 LLM 기반 AI Agent 구조를 기획하고 있습니다.

🖥️ Project 01

인터랙티브 아바타 서비스 기획

사진 한 장으로 만든 아바타가 대화까지 — AI Agent 구조 설계 프로젝트

사진 또는 짧은 영상만으로 생성된 아바타를 대화 가능한 AI 휴먼으로 확장한 B2B 서비스

기존 가상인간 제작은 비용과 시간이 많이 소요되며, 대화 기능이 없어 정적인 콘텐츠 소비에 머무르는 한계가 존재

이를 해결하기 위해 LLM + RAG 기반 구조를 통해 대화 가능한 인터랙티브 아바타 서비스로 확장

실제 작업 화면
Image / Video Placeholder 프로젝트 스크린샷 또는 데모 영상

이 프로젝트는 하나의 질문에서 시작했습니다.

⚠️ Problem

왜 기존 구조에서는
대화가 불가능한가

01

생성 중심 구조

가상인간은 콘텐츠 생성에 초점이 맞춰져 있으며, 대화 기능이 부재

02

맥락 반영 불가

사용자 상황과 맥락을 반영하지 못하는 정적 콘텐츠 구조

03

인터랙션 단절

일방향 소비 구조로 사용자 참여와 피드백이 차단됨

구조적 해결 방법을 고민했습니다.

💡 Solution — AI Agent Approach

LLM과 RAG 구조를 활용하여
맥락 기반 응답을 생성하는
대화형 구조 설계

사용자 입력을 이해하고, 맥락을 검색하며, 상황에 맞는 응답을 생성하는 구조적 AI 의사결정 프로세스
