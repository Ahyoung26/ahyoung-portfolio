What Kind of Planner

사용 흐름·구조로 문제를 정리하는 기획자

기능 나열이 아니라 진입–생성–대화–검증까지 한 축으로 묶어 설명 가능하게 만드는 일을 합니다.

핵심 요약

  • 이탈·행동 데이터를 바탕으로 UX·정책·데이터 구조를 같이 맞춤

핵심 요약

  • 생성 중심 인식에서 대화·실시간 상호작용 축으로 전환

3. Why AX

AI를 기능 목록이 아니라 사용·운영 흐름으로 묶은 이유

  • AX는 모델 탑재가 아니라 진입–생성–설정–대화–검증 단위를 재설계하는 일.
  • 파일·URL·RAG는 스펙 나열이 아니라 대화 맥락을 확장하는 선택적 장치로 정의.

의사결정 축 요약

축판단 질문
경험사용자가 「대화한다」고 느낄 만큼 빠른가
구조대화형이 제품 안에서 독립적으로 설명 가능한가
확장지식 주입(파일/URL)이 선택적으로 붙는가
운영B2B 반복 사용·비용 제약 안에서 지속 가능한가

4. Solution

아바타 역할 재정의 · 독립 생성·대화 UX 정렬

핵심 요약 (변화)

  • 아바타 = 산출물 → 대화 인터페이스
  • 종속 흐름 제거 → 독립 생성·프로젝트 단위
  • 기본 GPT-4o-mini · 자료 첨부 시 자사 RAG · 필수/확장 경계를 기획 문서로 먼저 분리
  • RAG: 자사 구축 RAG 연결 · 파일 우선 적용, URL은 구조상 수용
  • OCR: PDF·Word를 우선 적용 범위로 정의
  • 고도화 분리: 제스처·추가 표현은 당장 필수가 아닌 로드맵 항목으로 분리

5. Workflow

사용자 흐름 · 기본 LLM과 선택적 RAG 분기

파일 없이도 대화 가능. 자료가 붙으면 문서 맥락이 더해지는 구조입니다.

핵심 요약

  • 업로드 없음 → 기본 LLM만
  • 파일/URL 있음 → OCR(PDF·Word) → RAG 합류
  • 아래 다이어그램 = 면접관용 30초 플로우 스냅샷
  1. 진입 — 서비스/프로젝트 진입
  2. 아바타 생성 — 웹캠 또는 파일 업로드
  3. 프로젝트 화면 — 생성 결과 기반 작업 공간
  4. 언어·프롬프트 — 대화 설정
  5. 선택 — 파일 또는 URL 업로드(없으면 기본 LLM만)
  6. 대화 — 일반 대화 또는 문서 기반 응답

분기: 업로드 없음 → 기본 LLM 대화만. 파일/URL 추가 → 자사 RAG(문서 처리 시 OCR: PDF·Word)로 맥락 반영.
